## 执行步骤

```bash
cd /Users/popmart/Documents/python/forge-agent

cat > CHANGELOG.md <<'EOF'
# Changelog

All notable changes to forge-agent will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.3.0] - 2026-06-27

### Added
- Unified logging system (structlog + contextvars)
- `forge-agent logs` command (tail/follow/json)
- 自动注入 agent_id / run_id / domain 到 contextvars
- 双 renderer：dev console / prod JSON (FORGE_LOG_JSON=1)
- stdlib bridge：第三方库日志也走统一格式
- 异步任务隔离：并发 agent 互不串 contextvars
- 19 个新测试

### Changed
- `StdLogger` 改为 structlog 后端薄包装
- `BaseAgent.run()` 自动 bind contextvars

## [0.2.0] - 2026-06-XX

### Added
- LLM-driven code generator (4 layers: secrets/config/protocol/registry)
- 5 LLM providers: OpenAI, Anthropic, Gemini, Ollama, Mock
- CodeStore with versioning (v1/v2/...)
- MANIFEST.json 原子写 + 文件锁
- 15 CLI commands

## [0.1.0] - 2026-06-XX

### Added
- BaseAgent 强契约（3 必填 + 5 可选能力）
- 5 capabilities: log / search / memory / reflect / prompt_manager
- AgentRegistry 单例
- Scheduler（顺序/并行/优先级 3 策略）
- Pipeline DAG 编排
EOF

git add CHANGELOG.md
git commit -m "docs: add CHANGELOG.md"
```
