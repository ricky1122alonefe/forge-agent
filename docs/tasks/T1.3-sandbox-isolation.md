## 目标

把当前 `Sandbox` 从"in-process AST 检查 + asyncio timeout"升级为**真隔离**：

1. **进程隔离** — 在 subprocess 跑生成代码，不在主进程直接 import
2. **资源限制** — CPU/内存/文件大小/网络都要管
3. **文件系统隔离** — 写文件只在临时目录，不能动主目录
4. **网络隔离（默认 deny）** — 默认禁外网，需要时显式 allowlist
5. **崩溃兜底** — 子进程 OOM / 死循环 / 抛异常，都要回收 + 报告

**当前状况**（v0.2）：
- `src/forge_agent/generator/sandbox.py` 只有 AST import 检查 + `asyncio.wait_for` timeout
- 跟 validator 重叠（都在查 import 黑名单）
- 进程内跑，LLM 写的代码一旦 `import subprocess` 然后 `os.system("rm -rf /")` 真会执行

**为什么必须做**：
- 现在 validator 只查 import，但 LLM 可能在 `from xxx import yyy` 里偷换路径绕过
- 进程内跑 = 沙箱代码能访问主进程内存、网络、文件系统
- 没有内存/CPU 限制 = 死循环吃光资源
- 这两条不修，Phase 2 跑 MCP 真工具会出事

## 执行步骤

### Step 1: 重构 `generator/sandbox.py`

拆成两层：
- `StaticCheck` — 纯静态检查（AST），独立可用，给 quick-reject
- `SubprocessRunner` — 真隔离的子进程执行器

```python
# 顶层 API 不变，保持向后兼容
class Sandbox:
    def __init__(self, limits: ResourceLimits | None = None): ...
    async def run_smoke_test(self, agent_cls, sample_context) -> SmokeTestResult: ...
```

内部用 `SubprocessRunner` 跑：

```python
class SubprocessRunner:
    def __init__(self, limits: ResourceLimits): ...
    def run(self, source: str, entry: str, args: dict) -> RunResult: ...
```

### Step 2: 子进程执行模型

- 用 `multiprocessing.Process` 或 `subprocess.run` 隔离（选 `subprocess`，更标准）
- 子进程入口写一个 `subprocess_entry.py` 脚本：
  - 接收 source code + entry 函数名 + 参数（通过 stdin JSON）
  - 在子进程里 `exec(source, globals())` 拿到类
  - `asyncio.run(agent.run(**args))` 跑
  - 把结果 / 异常 / traceback 序列化成 JSON 输出到 stdout
  - 主进程读 stdout 拿结果
- **超时就 SIGKILL 整个子进程树**

### Step 3: 资源限制

跨平台用 `resource` 模块（Unix）：
```python
import resource
resource.setrlimit(resource.RLIMIT_CPU, (cpu_seconds, cpu_seconds))
resource.setrlimit(resource.RLIMIT_AS, (memory_mb * 1024 * 1024,) * 2)
resource.setrlimit(resource.RLIMIT_FSIZE, (file_size_mb * 1024 * 1024,) * 2)
```

macOS `RLIMIT_AS` 限制虚拟内存，足够防 OOM。

### Step 4: 文件系统隔离

- 主进程创建一个 `tempfile.mkdtemp()` 临时目录
- 子进程 `chdir` 到该目录
- **不挂载 bind mount**（macOS 太复杂），但写文件只能在该目录里
- 通过 `ResourceLimits.file_write: bool` 控制，False 时挂 `ReadOnlyFilesystem` 错误

### Step 5: 网络隔离（默认 deny）

- 简单做法：**不真做** socket 拦截（太复杂）
- 用 AST 检查：扫描 `socket` / `httpx` / `urllib` / `aiohttp` 调用
- 在 `NetworkPolicy` 里配置 allowlist：
  - `allow_localhost: bool = True`（127.0.0.1）
  - `allow_egress: list[str] = []`（显式白名单域名）
- 检查到不允许的网络调用 → 静态拒

### Step 6: 写测试 `tests/unit/test_sandbox.py`

覆盖：
- ✅ 子进程能正常跑通 agent
- ✅ 死循环（`while True:`）被 SIGKILL + 报 timeout
- ✅ 内存爆炸（`x = 'a' * 10**9`）被 RLIMIT_AS 干掉
- ✅ `os.system("echo hi")` 被子进程允许（子进程有自己的 resource），但被 AST 检查拒
- ✅ 写 `/tmp/evil.txt` 可以（在子进程的 cwd），但写主进程的工作目录被 chdir 隔离
- ✅ 网络调用 `httpx.get("https://api.x.com")` 被 NetworkPolicy 拒（默认 deny）
- ✅ 旧 API（`Sandbox().run_smoke_test(...)`）向后兼容
- ✅ 已有测试不挂

## 实施备忘

- **Python 3.14 注意**：`subprocess` 子进程的 `RLIMIT_AS` 在 Linux/macOS 行为不同，测试要用 `pytest.mark.skipif(sys.platform == "win32")`
- **临时目录清理**：用 `tempfile.TemporaryDirectory()` 上下文管理器，跑完自动清
- **超时粒度**：CPU 限制是累计 CPU 秒，asyncio timeout 是墙钟秒，两者都设
- **不要复用 `multiprocessing`**：它和 asyncio 集成有坑，用 `subprocess.Popen` 跑 `python -c` 更直接
