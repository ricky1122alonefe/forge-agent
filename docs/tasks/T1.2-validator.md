## 目标

把现在只查"类存在/3 方法有"的 validator，**强化**到能：
1. **Import 黑名单** — 拒 `os.system` / `subprocess` / `shutil.rmtree` 等
2. **复杂度上限** — cyclomatic complexity ≤ 10
3. **类型注解** — 所有方法必须有 return type
4. **危险模式检测** — 任意路径的 `open(..., "w")` 必须有安全前缀
5. **网络白名单** — 显式标注的 `httpx` / `aiohttp` 调用必须明确 URL

---

## 执行步骤

### Step 1: 增强 `validator.py`

在 `src/forge_agent/generator/validator.py` 加：

```python
DANGEROUS_IMPORTS = {
    "os.system", "subprocess", "shutil.rmtree",
    "eval", "exec", "__import__",
}

DANGEROUS_PATTERNS = [
    r"open\([^)]*['\"]w['\"]",       # 文件写入
    r"\.rmtree\(",
    r"\.system\(",
]
```

### Step 2: 加复杂度计算

用 `ast` 算每个函数的 cyclomatic complexity（条件/循环/异常）。

### Step 3: 加类型注解检查

每个方法必须有 `returns` 注解。

### Step 4: 写测试 `tests/unit/test_validator.py`

- 拒 `os.system("rm -rf /")`
- 拒 `open("/etc/passwd", "w")`
- 拒复杂度 > 10
- 拒缺类型注解的方法
- 已有测试不破

### Step 5: 验证

```bash
pytest tests/unit/test_validator.py -v
pytest tests/unit/test_generator.py -v  # 现有测试不挂
```
