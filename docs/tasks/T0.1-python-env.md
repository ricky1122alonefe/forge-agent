## 目标

在本地或 CI 环境中安装 Python 3.10+（推荐 3.11），确保所有后续测试在正确的 Python 版本上跑。

---

## 为什么需要这个

- `pyproject.toml` 声明 `requires-python = ">=3.10"`，但当前环境是 Python 3.9.6
- 3.10+ 的特性（`X | None`、`match`、精确类型提示）已大量使用
- 所有 v0.3 之前的测试都在 3.9 跑过，**没真验证过 3.10+**

---

## 执行步骤

### 方案 A：pyenv（推荐）

```bash
brew install pyenv
pyenv install 3.11.7
cd /Users/popmart/Documents/python/forge-agent
pyenv local 3.11.7
python --version  # 应显示 3.11.7
```

### 方案 B：Docker

```bash
docker pull python:3.11-slim
docker run -it --rm -v $(pwd):/app -w /app python:3.11-slim bash
# 容器内
python --version
```

### 方案 C：brew 直接装

```bash
brew install python@3.11
python3.11 --version
```

---

## 验证标准

- [ ] `python --version` 显示 3.10+（推荐 3.11）
- [ ] `python -c "import sys; print(sys.version_info[:2] >= (3, 10))"` 输出 `True`
- [ ] `pip install -e ".[dev]"` 干净通过（无版本警告）
- [ ] `python -c "import forge_agent; print(forge_agent.__version__)"` 输出 `0.3.0`

---

## 自动化验证命令

```bash
python --version | grep -E "3\.(1[0-9]|[2-9][0-9])" && echo "T0.1 PASS"
```

或在 `scripts/check_phase0.sh` 中：

```bash
python_version=$(python -c "import sys; print(f'{sys.version_info.major}.{sys.version_info.minor}')")
if python -c "import sys; sys.exit(0 if sys.version_info >= (3, 10) else 1)"; then
    echo "[T0.1] Python 3.10+ 真环境        ✅ PASS ($python_version)"
else
    echo "[T0.1] Python 3.10+ 真环境        ❌ FAIL (got $python_version, need >=3.10)"
    exit 1
fi
```
