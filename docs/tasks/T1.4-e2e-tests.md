## 目标

验证 **完整生成链路**（generate → validate → sandbox → deploy）在多个领域上跑通：

1. **10 个领域** — stock / football / weather / news / crypto / ecommerce / social / health / education / generic
2. **每个领域** — 生成 agent 代码 → validator 通过 → sandbox 跑通 → 产出 AgentReport
3. **真实 LLM** — 使用 DeepSeek provider（不是 mock）
4. **失败回滚** — 如果某个领域失败，不影响其他领域

---

## 执行步骤

### Step 1: 创建 `tests/e2e/test_generation_e2e.py`

用 `pytest.mark.parametrize` 跑 10 个领域：

```python

    ("stock", "monitor AAPL stock price"),
    ("football", "predict Premier League match outcome"),
    ("weather", "forecast tomorrow's weather in Beijing"),

    ("crypto", "track Bitcoin price movements"),
    ("ecommerce", "recommend products based on browsing history"),
    ("social", "analyze Twitter sentiment"),
    ("health", "track daily step count and calories"),
    ("education", "generate quiz questions for Python basics"),
    ("generic", "process and categorize customer feedback"),
]

@pytest.mark.parametrize("domain,prompt", DOMAINS)
async def test_generate_and_run(domain: str, prompt: str) -> None:
    # 1. generate agent code
    # 2. validate with ContractValidator
    # 3. run in Sandbox
    # 4. assert AgentReport is valid
```

### Step 2: 用真实 DeepSeek API

- 设置 `DEEPSEEK_API_KEY` 环境变量（CI 用 secrets）
- 本地跑时跳过（`pytest.mark.skipif`）
- 每个领域生成独立 agent，不共享状态

### Step 3: 验证标准

每个领域必须：
- ✅ `validation_result.ok == True`
- ✅ `sandbox_result.success == True`
- ✅ `sandbox_result.report` 是有效的 AgentReport dict
- ✅ `report["verdict"]` 在 Verdict 枚举里
- ✅ `report["agent_id"]` 非空

### Step 4: 失败隔离

- 每个领域独立测试，一个失败不影响其他
- 用 `pytest.mark.xfail` 标记已知不稳定的领域
- 失败时打印完整错误（validator errors + sandbox stderr）

### Step 5: 性能基线

记录每个领域的：
- 生成时间（LLM 调用）
- 验证时间
- sandbox 执行时间
- 总时间

写入 `tests/e2e/performance_baseline.json`（可选）
